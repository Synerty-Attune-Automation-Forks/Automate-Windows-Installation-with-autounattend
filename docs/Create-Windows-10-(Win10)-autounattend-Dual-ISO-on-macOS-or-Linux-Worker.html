Step 2 - Check Maximum Length of Win10 Hostname on macOS or Linux Worker

Windows Desktop 10 has a hostname maximum length of 15 characters.

if one of these Attune parameters are true, this step checks for the hostname length and fails if it is larger than 15. 1. isWin10Bios 2. isWin10Uefi

Execute the following script: 

                
if {isWin10Bios} || {isWin10Uefi}
then

    HOSTNAME="{newOsNode.hostname}"
    
    LENGTH="${#HOSTNAME}"
    MAX_LENGTH="15"
    
    if [ "${LENGTH}" -gt "${MAX_LENGTH}" ]
    then
        echo "Length of hostname ${LENGTH} for Win10 can't be more than 15."
        false
    else
        echo "Length of hostname ${LENGTH} is less than or equal to 15."
    fi
    
else
    echo "Not Win10, skipping."
fi

Step 3 - Check if IP Address is in Subnet on macOS or Linux Worker

To Debug

  • Uncomment out the echos to debug.

Logic

  • For loop loops through each octet in the IP address, subnet range octet and subnet mask.
  • For each octet:
    • The min octet is the subnet octet bitwised ANDed with the netmask octet.
    • A wildcard octet is 255 - netmask octet.
    • The max octet is the subnet octet bitwised ORed with the wildcard octet.
    • The IP address octet is tested to see if it is between the min and max octets.

References

Subnet Cheat Sheet - 24 Subnet Mask, 30, 26, 27, 29, and other IP Address CIDR Network References

Execute the following script: 

                
echo "IP address={newOsNode.ip}"
echo "subnet={newOsNodeSubnet.subnet}"
echo "netmask={newOsNodeSubnet.netmask}"
#echo "i, IP_OCTET, SUBNET_OCTET, NETMASK_OCTET, WILDCARD_MASK, OCTET_MIN, OCTET_MAX"

for i in $(seq 1 1 4)
do
    IP_OCTET=$(echo "{newOsNode.ip}" | awk -F'.' "{print \$$i}")
    SUBNET_OCTET=$(echo "{newOsNodeSubnet.subnet}" | awk -F'.' "{print \$$i}")
    NETMASK_OCTET=$(echo "{newOsNodeSubnet.netmask}" | awk -F'.' "{print \$$i}")
    
    OCTET_MIN=$((SUBNET_OCTET & NETMASK_OCTET))

    WILDCARD_MASK=$((255 - NETMASK_OCTET))
    OCTET_MAX=$((SUBNET_OCTET | WILDCARD_MASK))
    
    #echo "$i. $IP_OCTET, $SUBNET_OCTET, $NETMASK_OCTET, $WILDCARD_MASK, $OCTET_MIN, $OCTET_MAX"
    
    if [ "$IP_OCTET" -ge "$OCTET_MIN" ] &&
        [ "$IP_OCTET" -le "$OCTET_MAX" ]
    then
        echo "Octet number $i, $OCTET_MIN <= $IP_OCTET <= $OCTET_MAX, ok."
    else
        echo "Octet number $i, $OCTET_MIN <= $IP_OCTET <= $OCTET_MAX is false. Not in range of the subnet mask."
        false
    fi
done

Step 7 - Extract Windows boot.img on macOS or Linux Worker

Extracts the boot.img from the ISO. This is required for passing to mkisofs to regenerate a bootable BIOS ISO.

Only applies to BIOS kickstarts so if any of these parameters are set the string true:

  1. isWin10Bios
  2. isWinServerBios

Execute the following script: 

                
if {isWin10Bios} || {isWinServerBios}
then

    # Extract the boot.img out of the WinPE ISO
    
    cd {automationWorkerLinuxBaseDirectory}/build-{newOsNode.fqn}
    
    # Case insensitve globbing
    shopt -s nocaseglob
    
    # Set the filename of the ISO
    F=$(ls *.iso)
    
    # Get the Boot image offset
    OFFSET=`isoinfo -d -i $F | grep Bootoff | cut  -c21-`
    
    # Extract the boot.img.
    dd if=$F of=boot.img bs=2048 count=8 skip=$OFFSET
    
else
    echo "Skipping for UEFI ISO creation."
fi

Step 10 - Create BIOS Boot ISO on macOS or Linux Worker

Creates a BIOS bootable Windows ISO called kickstart_{newOsNode.fqn}.iso from the contents in folder {automationWorkerLinuxBaseDirectory}/build-{newOsNode.fqn}.

Only applies BIOS kickstarts so if any of these parameters are set the string true'`:

  1. isWin10Bios
  2. isWinServerBios

Execute the following script: 

                
cd "{automationWorkerLinuxBaseDirectory}/build-{newOsNode.fqn}"

if {isWin10Bios} || {isWinServerBios} 
then
    if uname | \
        grep -iq "darwin"
    then
        echo "is macOS"
        export PATH="${HOME}/Parallels/homebrew/bin:$PATH"
        echo "Using $(which xorriso)"
        
        xorriso -as mkisofs \
            -b boot.img \
            -no-emul-boot \
            -c BOOT.CAT \
            -iso-level 3 \
            -udf \
            -joliet-long \
            -relaxed-filenames \
            -J \
            -full-iso9660-filenames \
            -disable-deep-relocation \
            -omit-version-number \
            -V "KS_WIN" \
            -o {automationWorkerLinuxBaseDirectory}/kickstart_{newOsNode.fqn}.iso \
            .
            
    else
        echo "not macOS"
        
        mkisofs \
            -b boot.img \
            -no-emul-boot \
            -c BOOT.CAT \
            -iso-level 3 \
            -udf \
            -joliet-long \
            -relaxed-filenames \
            -allow-limited-size \
            -J \
            -full-iso9660-filenames \
            -disable-deep-relocation \
            -omit-version-number \
            -V "KS_WIN" \
            -o {automationWorkerLinuxBaseDirectory}/kickstart_{newOsNode.fqn}.iso \
            .
    fi
        
else
    echo "Skipping for UEFI boot."
fi

Step 11 - Create UEFI Boot ISO on macOS or Linux Worker

Creates a UEFI bootable Windows ISO called kickstart_{newOsNode.fqn}.iso from the contents in folder {automationWorkerLinuxBaseDirectory}/build-{newOsNode.fqn}.

Only applies UEFI kickstarts so if any of these parameters are set the string true:

  1. isWin10Uefi
  2. isWinServerUefi

https://unix.stackexchange.com/questions/531012/how-to-modify-an-installation-iso-and-keep-it-bootable

Execute the following script: 

                
if {isWin10Uefi} || {isWinServerUefi}
then
    cd "{automationWorkerLinuxBaseDirectory}/build-{newOsNode.fqn}"
    
    if uname | \
        grep -iq "darwin"
    then
        echo "is macOS"
        export PATH="${HOME}/Parallels/homebrew/bin:$PATH"
        echo "Using $(which xorriso)"
        
        xorriso -as mkisofs \
            -allow-limited-size \
            -iso-level 4 \
            -full-iso9660-filenames \
            -rock \
            -b boot/etfsboot.com \
            -no-emul-boot \
            -boot-load-seg 0 \
            -boot-load-size 8 \
            -hide boot.catalog \
            -eltorito-alt-boot \
            -no-emul-boot \
            -b efi/microsoft/boot/efisys.bin \
            -boot-load-size 1 \
            -udf \
            -untranslated-filenames \
            -disable-deep-relocation \
            -o {automationWorkerLinuxBaseDirectory}/kickstart_{newOsNode.fqn}.iso \
            .
            
    else
        echo "not macOS"
        
        mkisofs \
            -allow-limited-size \
            -iso-level 4 \
            -full-iso9660-filenames \
            -rock \
            -b boot/etfsboot.com \
            -no-emul-boot \
            -boot-load-seg 0 \
            -boot-load-size 8 \
            -hide boot.catalog \
            -eltorito-alt-boot \
            -no-emul-boot \
            -b efi/microsoft/boot/efisys.bin \

            -udf \

            -disable-deep-relocation \
            -o {automationWorkerLinuxBaseDirectory}/kickstart_{newOsNode.fqn}.iso \

Step 15 - Create Answer File ISO on macOS or Linux

Creates an ISO at {automationWorkerLinuxBaseDirectory}/drivers_{newVmNode.fqn}.iso containing:

  1. drivers folder
  2. autounattend.xml
  3. post_install_setup.ps1

This works for kickstarts on Linux or macOS with: 1. Windows Desktop 10 2. Windows Server 2016 2. Windows Server 2019 3. Windows Server 2022

Execute the following script: 

                
cd {automationWorkerLinuxBaseDirectory}/build-{newOsNode.fqn}

DRIVERS_FOLDER="drivers"
DRIVERS_ISO_NAME="drivers_{newOsNode.fqn}.iso"

if uname | \
    grep -iq "darwin"
then
    echo "is macOS"
    
    export PATH="${HOME}/Parallels/homebrew/bin:$PATH"
    echo "Using $(which xorriso)"
    xorriso -as mkisofs -o ../${DRIVERS_ISO_NAME} \
        -iso-level 2 -joliet-long -relaxed-filenames \
        -J -l -D -N -V "DRIVERS" "${DRIVERS_FOLDER}" \
        autounattend.xml post_install_setup.ps1
else
    echo "not macOS"
    
    mkisofs -o ../${DRIVERS_ISO_NAME} \
        -iso-level 3 -udf -joliet-long -relaxed-filenames \
        -J -l -D -N -V "DRIVERS" "${DRIVERS_FOLDER}" \
        autounattend.xml post_install_setup.ps1
fi
